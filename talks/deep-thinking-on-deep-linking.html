how deeplinks are handled android

  1. Open the user's preferred app that can handle the URI, if one is designated.
  2. Open the only available app that can handle the URI.
  3. Allow the user to select an app from a dialog.

PROMPT: android deeplink basics

android doc: https://developer.android.com/training/app-links/deep-linking

<activity
    android:name="com.example.android.GizmosActivity"
    android:label="@string/title_gizmos" >
    <intent-filter>
      ...
      <data android:scheme="https" android:host="www.example.com" />
      <data android:scheme="app" android:host="open.my.app" />
    </intent-filter>
    <intent-filter android:autoVerify="true">
        <action android:name="android.intent.action.VIEW" />
        <category android:name="android.intent.category.DEFAULT" />
        <category android:name="android.intent.category.BROWSABLE" />
        <data android:scheme="http" />
        <data android:scheme="https" />
        <data android:host="www.example.com" />
        <data
            android:host="www.example.com"
            android:pathPattern="/.*\.cp" />
        <data
            android:host="www.example.com"
            android:pathPattern="/.*\.ip" />
    </intent-filter>
</activity>



override fun onCreate(savedInstanceState: Bundle?) {
    super.onCreate(savedInstanceState)
    setContentView(R.layout.main)
    val action: String? = intent?.action
    val data: Uri? = intent?.data
}

test the deeplink...

$ adb shell am start
        -W -a android.intent.action.VIEW
        -d "example://gizmos" com.example.android

Sample 1

test:

adb shell am start -W -a android.intent.action.VIEW -d "computeristsolutions://first" com.example.deeplinksample1 adb shell am start -W -a android.intent.action.VIEW -d "computeristsolutions://second" com.example.deeplinksample1

talking points:

  • no backstack
  • intent filter on each activity
  • simple and by the book
  • gets harder to maintain as number of deeplinks increases

Sample 2

test:

adb shell am start -W -a android.intent.action.VIEW -d "computeristsolutions://first" com.example.deeplinksample2 adb shell am start -W -a android.intent.action.VIEW -d "computeristsolutions://second" com.example.deeplinksample2

talking points:

  • solves backstack in this example because we use the Main activity; doesn't work for all cases
  • centralized routing easier to maintain
  • dynamic routing would be better
    • limit: still must be defined in manifest
  • cold start vs warm start
  • user attribution is useful information
