my goal for this talk

flim-flam, hand-wavy, acronym laced, self-help guru style ranting.. to briefly cover a couple bits about Kotlin, and maybe get you to think about using it if you're not already.

A few Kotlin tidbits and highlights

  • runs on JVM
  • Industry driven for the purpose of productivity
  • no runtime cost (its just bytecode)
  • expressive and concise
  • java interop
  • supports functional style coding
  • coroutines for async
  • co-exist in your existing Java app

1. It's just JVM!

  • high rate of adoption on android
  • but its not just for android
  • runs anywhere the jvm runs, afaik
  • server, android, web...
  • plus multi-platform native (ios and android with Kotlin Native)

2. Made with love

  • from JetBrains because they like the JVM but want to move faster
  • purpose built for productivity with best features from multiple languages
  • modernize features beyond what Java provides
  • intentional interop
  • developer happiness
  • better generics

3. Nullability, yeah, yeah...

  • inferred types:

    val name = "josh" // because it's obviously a string

  • Kotlin collections/arrays/etc are Java collections/arrays/etc. simpler & better then inventing it's own

  • Kotlin implements a lot of collection helpers like listOf(..) or hashMapOf(..), everything (and more) you find in Collections class.

  • extension methods:

    fun BigDecimal.percent(percent: Int) = this.multiply(BigDecimal(percent).divide(BigDecimal(100))) val myPercent = myMoney.amount.percent(33)

  • ..which is now a lot of the Kotlin standard library works

  • you can extend operators, but there be dragons

  • oh, yeah. nullability has to be explicit which avoids a large portion of null ref errors.

    val name: String = "josh" // cannot be null val name: String? = null // can be null and has to be checked before reading value

4. Simpler classes,

data class Money(val amount: BigDecimal, val currency: String) {
    fun getFormattedAmount() = "${currency}${amount}"
}


fun sendMoney(money: Money, message: String = "") { .. }
  • Kotlin complier generates the properties/getters/setters from the constructor
  • default parameters to reduce overload functions

  • can also define the constructor separately:

    class NewMoney { private lateinit var myMoney: Money constructor (money: Money) { myMoney = money } }

  • lateinit is promise to provide a value later

  • delegate values by lazy for lazy initialization

    val myKidsAre: String by lazy { // do something here }

5. A couple notes

  • it's really easy to start and, then forget to use semicolons when you editing a Java file
  • you will start writing java in Kotlin
  • you should learn standard library functions, and Kotlin idioms
